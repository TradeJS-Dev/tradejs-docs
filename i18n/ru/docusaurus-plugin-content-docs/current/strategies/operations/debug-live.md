---
title: Как отлаживать стратегию в live-режиме
---

Если стратегия ведет себя странно в live, разбирайте проблему по слоям.

## 1. Проверка входных данных

- Правильные `symbol`, `interval`, конфиг стратегии из Redis.
- Актуальные рыночные данные по символу и BTC-контексту.

## 2. Проверка решения стратегии

- Что вернул `core.ts`: `skip`, `entry` или `exit`.
- Если есть `entry`, но нет ордера — смотрите runtime policy и gate-результаты.

## 3. Проверка runtime-гейтов

- порог AI (`minQuality`)
- ML threshold / enrichment
- флаг `MAKE_ORDERS`

## 4. Проверка артефактов

- сигнал в Redis
- `analysis:<symbol>:<signalId>`
- логи за тот же таймстемп

## 5. Локальное воспроизведение

- Запускайте один символ и одну стратегию.
- Для повторяемости используйте `--cacheOnly`, когда это возможно.
